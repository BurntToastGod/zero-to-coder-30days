## 🧠 How to Approach a Project

Before writing a single line of code, do this:

1. **Describe it in plain English** — What does it do? What does the user see?
2. **Break it into pieces** — What are the small steps?
3. **Write pseudocode** — Fake code in English that describes each step
4. **Build one piece at a time** — Don't try to write everything at once
5. **Test as you go** — Run it after every small addition

## 🛠️ Project 1 — Calculator App (Days 23–24)
See: [`/projects/project-1-calculator/`](../projects/project-1-calculator/README.md)

**What it does:** A text-based calculator that can add, subtract, multiply, and divide.

**Concepts used:** variables, input, conditionals, functions, loops

**Core logic:**
```
1. Ask user for first number
2. Ask user what operation (+, -, *, /)
3. Ask user for second number
4. Calculate result
5. Print result
6. Ask if they want to calculate again
```

## 🏁 After Day 30 — What's Next?

### How to Stop Relying on Tutorials

1. **Build something you WANT** — motivation beats any tutorial
2. **Start with a tiny version** — make it work ugly before making it work pretty
3. **Use docs, not tutorials** — [docs.python.org](https://docs.python.org) is your new best friend
4. **Google specific errors** — copy the error message, paste it into search
5. **Read other people's code** on GitHub — you'll learn more than any course

### How to Build Your Own Projects from Scratch

The formula:
```
1. Pick something annoying in your life you want to automate or improve
2. Describe it in ONE sentence
3. Break it into 5 small steps
4. Write each step as a function
5. Connect the functions
6. Test and fix
```
